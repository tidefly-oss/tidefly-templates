# Security Policy

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities via GitHub Issues.**

Report vulnerabilities responsibly via [GitHub Private Security Advisories](https://github.com/tidefly-oss/tidefly-templates/security/advisories/new).

### What to Expect

- **Acknowledgement** within 48 hours
- **Status update** within 7 days
- **Credit** in the release notes (if desired)

## Scope

**In scope:**

- Templates that expose sensitive data unintentionally
- Templates that could lead to container escape or privilege escalation
- Templates with insecure default configurations

**Out of scope:**

- Vulnerabilities in the services themselves (report to the upstream project)
- Vulnerabilities in self-hosted infrastructure (your own servers)
